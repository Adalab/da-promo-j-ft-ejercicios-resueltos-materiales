Manejo de Ramas en Git 🌿

En este ejemplo, vamos a aprender cómo crear y manejar ramas en Git utilizando la rama ramita. ¡Empecemos! 🚀

Crear y Verificar Ramas 🌱

Primero, listamos las ramas actuales:

git branch

Luego, creamos una nueva rama llamada ramita:

git checkout -b ramita

Verificamos nuevamente las ramas para asegurarnos de que ramita fue creada: 

git branch

Cambiar a la Nueva Rama 🚶‍♀️

Cambiamos a la nueva rama ramita:

git checkout ramita

Ahora estamos trabajando en la rama ramita.

Subir la Rama al Repositorio Remoto 🌐

Para subir la nueva rama al repositorio remoto, usamos el siguiente comando (la primera vez puede dar error):

git push --set-upstream origin ramita

Hacer Cambios en la Rama ✏️

Realizamos algunos cambios en los archivos y luego los añadimos:

git add .

Hacemos un commit de los cambios:

git commit -m "Add cambios ramita"

Finalmente, subimos los cambios al repositorio remoto:

git push

Fusionar la Rama con main 🌟

Para fusionar ramita con main, primero cambiamos a la rama main:

git checkout main

Verificamos las ramas:

git branch

Actualizamos la rama main con los cambios remotos:

git pull

Fusionamos ramita con main:

git merge ramita

Finalmente, subimos los cambios fusionados al repositorio remoto:

git add . + git commit -m"" + git push

Eliminar una Rama 🗑️

Para eliminar una rama local que ya no necesitamos:

git branch -d ramita

Para eliminar una rama remota:

git push origin --delete ramita

Resolver Conflictos de Fusión ⚔️

Si encuentras un conflicto durante una fusión, Git te notificará. Para resolver conflictos:

- Edita los archivos en conflicto para solucionar los cambios manualmente.

- Marca los conflictos resueltos añadiendo los archivos:

git add [archivo_conflictivo]

- Termina la fusión con un commit:

git commit

Rebase de Ramas 🔄

Para rebasar la rama ramita sobre main:

git checkout ramita
git rebase main

Sube los cambios después de rebasar:

git push --force-with-lease

El rebase reescribe el historial de la rama, por lo que es importante hacerlo solo en ramas locales o privadas.

🔍 Ver las Ramas en el Repositorio 🔍

Para ver las ramas que hemos creado localmente:

git branch

Para ver las ramas remotas:

git branch -r

Para ver todas las ramas (locales y remotas):

git branch -a

Las ramas remotas aparecerán en rojo porque no son nuestras directamente.
